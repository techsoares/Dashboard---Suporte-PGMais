📊 Visao Geral

O que e o PGMais Dashboard e o que ele resolve.

O PGMais Dashboard e uma plataforma de gestao de demandas que integra o Jira Cloud com inteligencia artificial para dar visibilidade total sobre o fluxo de trabalho da equipe. Ele substitui a necessidade de acessar o Jira diretamente para tarefas do dia-a-dia.

⚡ Priorizacao Inteligente

Algoritmo de score que combina faturamento do cliente, tipo de issue, impacto em producao e avaliacoes por IA para determinar a ordem de atendimento.

🤖 IA Integrada

Assistente Claude Sonnet 4.6 disponivel em todas as telas. Analisa fila, historico, tendencias, SLA e recomenda acoes concretas com dados reais.

📈 Gestao Executiva

KPIs, graficos semanais, radar de desempenho por dev, analise de gargalos, mix de tipos e SLA — tudo com visao historica de ate 6 meses.

🔒 Controle de Acesso

Autenticacao JWT, dominio corporativo, painel admin para BUs, ranking de faturamento com edicao restrita a admins.

Dados em Tempo Real O dashboard sincroniza com o Jira a cada 5 minutos automaticamente. Voce tambem pode forcar atualizacao manual clicando no botao "↻ atualizar".

⚡ Priorizacao de Demandas

Fila inteligente baseada em faturamento, tipo, producao e idade.

Como funciona o Score

Cada issue recebe um score calculado automaticamente:

CriterioPontosDescricao
Producao afetada+1000IA classifica se o chamado impacta producao do cliente
Incidente/Bug+500Tipo de issue e incidente ou bug
Problema+300Tipo de issue e problema
Top Account+40 a +400Account no ranking de faturamento (Top 10): (11 - posicao) x 40
Idade+3/dia (max 200)Dias desde a criacao da issue
Boost de Prioridade+0 a +750Solicitacoes de prioridade avaliadas por IA

Ranking de Faturamento

Painel lateral mostra a ordenacao dos clientes por faturamento. Somente administradores podem alterar a ordenacao (subir, descer, adicionar ou remover accounts). Usuarios comuns visualizam o ranking como somente leitura.

Solicitar Prioridade

Qualquer usuario pode solicitar prioridade para um chamado:

  1. Clique no botao ⚡ ao lado do chamado (ou use o botao "⚡ Priorizar" no header)
  2. Preencha seu nome, BU e uma justificativa detalhada
  3. A IA avalia a urgencia e atribui um boost de 0 a 500 pontos
  4. BUs de gestao recebem multiplicador 1.5x (maximo 750 pontos)
  5. O chamado e reposicionado automaticamente na fila
Dica para boost alto Descreva o impacto real: producao parada, perda financeira, SLA estourado, cliente em risco de churn. Justificativas genericas como "e urgente" recebem boost baixo.

Despriorizacao (Admin)

Administradores podem remover qualquer solicitacao de prioridade individualmente (botao ✕ em cada request) ou todas de uma vez ("Despriorizar tudo") quando ha 2+ requests no mesmo chamado.

📋 Kanban

Board tradicional com colunas por status.

Colunas geradas automaticamente a partir dos status do Jira. Dentro de cada coluna, issues sao ordenadas por:

  1. Issues atrasadas primeiro (destaque vermelho "ATRASADO")
  2. Prioridade: Highest → Lowest

Cada card mostra: chave, titulo, tipo, responsavel, data de vencimento e badge de prioridade colorido.

IA na Priorizacao

Dois usos de IA na tela de priorizacao:

  • Classificacao de producao: IA analisa todas as issues e identifica quais afetam producao do cliente (+1000 pontos no score).
  • Avaliacao de boost: Ao solicitar prioridade, a IA avalia a justificativa e atribui de 0 a 500 pontos baseado no impacto real.

⚙️ Painel Administrativo

Gestao de BUs, admins e configuracoes do sistema. Acesso restrito.

Unidades de Negocio (BUs)

📁
Criar e gerenciar BUs
Crie BUs com nome e tipo (Operacional ou Gestao). BUs de gestao tem multiplicador 1.5x no boost de priorizacao e podem despriorizar chamados.
👥
Atribuir membros
Clique nos nomes para adicionar/remover de uma BU. Arraste para atribuicao rapida. Veja usuarios sem BU em secao separada.

Administradores

Na aba "Administradores", gerencie contas admin:

  • Adicionar novo admin (email + senha forte: min 8 chars, 1 maiuscula, 1 numero)
  • Alterar senha de admin existente
  • Remover admin (nao pode remover a si mesmo)
Permissoes de Admin Admins podem: editar ranking de faturamento, despriorizar qualquer solicitacao, gerenciar BUs/membros, adicionar/remover admins, forcar refresh do cache.

Integracoes Externas

Jira Cloud REST API v3

Autenticacao Basic Auth (email + API token). Busca issues ativas, concluidas, historico semanal. Cache de 5min para dashboard, 1h para gestao.

OpenRouter (Claude Sonnet 4.6)

API de IA generativa. Usada para: chat contextual, classificacao de producao, avaliacao de boost de prioridade. Custo por token.

Arquitetura

      v

🔒 Seguranca

Medidas de protecao implementadas no sistema.

🔐
Autenticacao JWT
Tokens com expiracao de 8h. Todos os endpoints sensiveis exigem token valido no header Authorization.
🔒
Senhas com bcrypt
Senhas de admin armazenadas como hash bcrypt. Validacao de complexidade: minimo 8 caracteres, 1 maiuscula, 1 numero.
🛡️
Rate Limiting
API geral: 100 req/min. Login: 15 tentativas em 15 min por IP. Previne brute force.
🔧
Headers de Seguranca
X-Content-Type-Options, X-Frame-Options, HSTS, CSP, Referrer-Policy em todas as respostas.
🌐
CORS restrito
Somente localhost e dominios configurados em ALLOWED_ORIGINS. Regex-based para GitHub Codespaces.
👤
Dominios permitidos
Login restrito a emails @pgmais.com.br e @ciclo. Auto-cadastro para dominio valido, senha obrigatoria para admins.

Como iniciar o projeto localmente? 
# Backend
cd backend
python -m venv .venv
.venv\Scripts\activate       # Windows
pip install -r requirements.txt
cp .env.example .env         # Configure suas credenciais
python main.py               # Sobe em http://localhost:8000

# Frontend
cd frontend
npm install
cp .env.example .env         # Aponte para o backend
npm run dev                  # Sobe em http://localhost:5173
